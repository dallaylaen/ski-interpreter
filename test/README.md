# Test structure

Please put the new tests into one of the categories below,
or create a new one and document it here:

* `core` - required to boot the module, or to run other tests
* `edge-cases` - edge cases and incorrect inputs
* `eval` - tests based on code samples and snippets
* `extras` - extra features not in the core
* `manage` - operations on the parser object: adding/removing terms, restricting, save/load etc.
* `quest` - quests
* `transform` - SKI<->Lambda, fancy formatting, and tree traversals
* `REAMDE.md` - this file
